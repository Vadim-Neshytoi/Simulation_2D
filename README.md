Simulation 2D
Описание проекта

Simulation 2D — это консольная пошаговая симуляция экосистемы на двумерной карте. 
В мире существуют различные объекты и живые существа, взаимодействующие друг с другом по заданным правилам. 
Травоядные ищут пищу и стараются избегать хищников, а хищники охотятся на травоядных для восстановления своих ресурсов.

Структура проекта

actions/ — действия, выполняемые во время симуляции
controller/ — обработка пользовательских команд
entities/ — сущности мира и существа
generation/ — генерация мира
map/ — хранение объектов и работа с картой
pathfinding/ — алгоритмы поиска пути
rendering/ — отображение мира
simulation/ — управление жизненным циклом симуляции

Архитектура построена вокруг нескольких основных компонентов:

Simulation

Координирует работу всей системы.

Отвечает за:

хранение состояния симуляции;
выполнение действий каждого тика;
обработку пользовательских команд;
запуск начальной генерации мира.

Map

Хранит все объекты и существа, находящиеся на карте.

Карта использует разреженное (sparse) представление данных. Внутренний словарь содержит только занятые клетки, 
что позволяет избежать хранения большого количества пустых значений.

Entity

Базовый класс всех сущностей мира.

От него наследуются:

Rock
Tree
Grass
Creature

Creature

Абстрактный базовый класс всех существ.

Содержит общие характеристики:

здоровье;
энергия;
состояние;
цель;
радиус обзора;
параметры отдыха и восстановления.

Наследники самостоятельно определяют своё поведение через реализацию метода make_move().

Herbivore

Травоядное существо.

Основные задачи:

поиск ближайшей травы;
перемещение к источнику пищи;
поедание травы;
бегство от ближайшего хищника;
восстановление энергии во время отдыха.
Predator

Хищник.

Основные задачи:

поиск ближайшей добычи;
преследование травоядных;
атака цели;
восстановление энергии после успешной охоты;
отдых при нехватке энергии.
Renderer

Отвечает за отображение текущего состояния мира в консоли.

Controller

Обрабатывает команды пользователя и управляет режимами работы симуляции.

Система действий (Actions)

Каждый тик симуляции состоит из последовательного выполнения действий.

Порядок выполнения:

DecisionAction
AttackAction
EatAction
NavigationAction
MoveCreaturesAction
EnergyAction
GrassRespawnAction
HerbivoreRespawnAction

Такое разделение позволяет изолировать различные этапы логики и соблюдать принцип единственной ответственности.

Поиск пути

Для построения маршрутов используется алгоритм поиска в ширину (Breadth-First Search, BFS).

Алгоритм применяется для определения следующего шага на пути к цели и позволяет существам обходить препятствия,
выбирая кратчайший доступный маршрут.

Система энергии

Каждое существо обладает ограниченным запасом энергии.

Энергия расходуется на:

перемещение;
атаку.

Энергия восстанавливается:

во время отдыха;
после поедания травы (травоядные);
после успешной охоты (хищники).

При снижении энергии ниже установленного порога существо переходит в режим отдыха. 
После восстановления достаточного количества энергии оно возвращается к обычному поведению.

Поведение существ
Травоядные

Травоядные постоянно анализируют окружающее пространство.

При обнаружении хищника они переключаются в режим бегства и стараются увеличить дистанцию до угрозы. 
При отсутствии опасности они ищут ближайшую доступную траву и направляются к ней.

Хищники

Хищники ищут ближайшую добычу в радиусе обзора. Если добыча найдена, хищник начинает преследование. 
При достижении дистанции атаки выполняется нападение.

После успешной охоты хищник получает дополнительное восстановление энергии.

Управление симуляцией

Доступные команды:

R — запуск симуляции;
P — пауза;
S — выполнить один шаг;
Q — завершить работу программы.
Запуск проекта

Windows (реальный тайм-режим)

На Windows используется неблокирующий ввод через msvcrt.

Особенности:

команды вводятся без нажатия Enter
симуляция работает в реальном времени
клавиши обрабатываются сразу после нажатия

Просто нажимайте клавиши r / p / s / q во время работы программы.

Linux / macOS (упрощённый режим ввода)

На Unix-системах используется стандартный ввод (input()).

Особенности:

после ввода команды нужно нажать Enter
симуляция работает в пошаговом режиме ввода команд
более простой, но кроссплатформенный вариант

Проект разработан и протестирован на Windows.
Код содержит кроссплатформенную реализацию ввода и должен работать на Linux/macOS при наличии Python 3.10+, 
однако данные платформы не тестировались.

Установка и запуск

Требования

Для запуска проекта необходимо:

Python 3.10 или новее
Git (только если планируется клонирование репозитория)

Проверить установленную версию Python можно командой:

python --version
Получение проекта
Вариант 1. Клонирование через Git
git clone <ссылка-на-репозиторий>
cd Simulation_2D
Вариант 2. Скачивание ZIP-архива
Откройте страницу репозитория на GitHub.
Нажмите кнопку Code.
Выберите Download ZIP.
Распакуйте архив в удобное место.
Запуск проекта

Перейдите в папку проекта и выполните:

python main.py

Если в системе используется несколько версий Python, может потребоваться команда:

python3 main.py
Управление симуляцией

Доступные команды:

R — запуск симуляции
P — пауза
S — выполнить один шаг симуляции
Q — выход из программы
Windows

Команды обрабатываются сразу после нажатия клавиши.

Linux / macOS

После ввода команды необходимо нажать Enter.
